## Purpose

Get from zero to a reproducible manifest workflow in a few commands using the reference CLI.

## Mental model

- `generate` creates the standardized contract artifact (`sondage.manifest.json`).
- `manifest` prints Sonde's own reference Sondage manifest.
- `run` validates deterministic behavior against that local contract.
- `score` outputs a compatibility report that can be compared across tools.

## Inputs

- Node.js and npm/pnpm installed.
- A target CLI available in `PATH` (recommended local fixture: [`cli-good`](https://github.com/sonde-sh/sonde/tree/main/examples/cli-good)).
- Write access in current working directory.

## Outputs

- `sondage.manifest.json` in current directory.
- Deterministic run output.
- Scoring output with a 0-100 score.

## Steps

### 1) Install or run directly

```sh
npx @sonde-sh/sonde --help
```

### 2) Generate manifest

```sh
sonde generate cli-good --json
```

Expected output shape:

```json
{
  "ok": true,
  "apiVersion": "1.0.0",
  "command": "generate",
  "cli": "cli-good",
  "result": {
    "manifest": {
      "version": "1.0.0"
    },
    "outputPath": "sondage.manifest.json",
    "commandCount": 2
  }
}
```

### 3) Inspect Sonde reference manifest

```sh
sonde manifest --json
```

Expected output shape:

```json
{
  "ok": true,
  "apiVersion": "1.0.0",
  "command": "manifest",
  "result": {
    "version": "1.0.0",
    "cli": {
      "name": "sonde",
      "binary": "sonde"
    }
  }
}
```

### 4) Run deterministic check

```sh
sonde run cli-good --json
```

Expected output shape:

```json
{
  "ok": true,
  "apiVersion": "1.0.0",
  "command": "run",
  "cli": "cli-good",
  "result": {
    "ok": true,
    "exitCode": 0
  }
}
```

### 5) Compute score

```sh
sonde score cli-good --json
```

Expected output shape:

```json
{
  "ok": true,
  "apiVersion": "1.0.0",
  "command": "score",
  "cli": "cli-good",
  "result": {
    "manifestVersion": "1.0.0",
    "generatedAt": "2026-03-05T00:00:00.000Z",
    "total": 100
  }
}
```

## Edge cases

- If target CLI is missing, generation fails with execution error.
- `run` and `score` require `sondage.manifest.json` in current directory.
- If CLI output looks interactive, run quality decreases and score may drop.
- For stable local docs/testing, prefer [`cli-good`](https://github.com/sonde-sh/sonde/tree/main/examples/cli-good) and compare against [`cli-ok`](https://github.com/sonde-sh/sonde/tree/main/examples/cli-ok) / [`cli-bad`](https://github.com/sonde-sh/sonde/tree/main/examples/cli-bad) to understand scoring trade-offs.
- Manifest loading fails with `UNSUPPORTED_VERSION` when manifest major version is not supported.
